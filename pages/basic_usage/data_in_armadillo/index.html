    Data in Armadillo

    This page aims to describe how data is kept and managed within Armadillo. Armadillo doesn't have an underlying database managing data, instead data is stored directly on the server on which armadillo is installed. Tabular data is stored in .parquet files and other files can be used and imported using the resourcer package.

    Tabular files

    Most data in armadillo, is stored in a tabular format. The file format we use for that is .parquet, a format that is designed to efficiently store and retrieve tabular data. To get your tabular data in Armadillo, either upload a csv/tsv (comma separated values or tab separated values) file using the Armadillo user interface, or use the Armadillo R Package.     Other file formats

    Armadillo technically supports all file formats. In order to allow researchers to use them however, they have to be made accessible. Because these files can be anything, it's harder to determine how to assign them in the researcher's environment. To address this problem, these files are made accessible as "resources", these are files that act like symlinks, pointing to where the actual file resides. An R package is used to create those "resources". This R package is called resourcer.

    Security

    In the scenario where tables are assigned in the researcher's environment, data is being pushed through the application, to the research environment. That way it is ensured, that data does not leave the application unless prompted. For other files, the process is slightly different. The R environment has to do a GET request to the application to assign data to the researcher's environment, this happens using a short-lived self-signed internal token. This token grants temporary access to these resources, if the requester is supposed to have access to the resource according to project permissions. The default lifespan for these tokens is 5 minutes (300 seconds), but can be adjusted in the application.yml, by setting the storage.resource-token-timeout property. This lifetime is the maximum time allowed between the assign.resource and assign.expr command when using resources. Once the assign.expr is done, the assigned resource is available for the entire duration of the researcher's session.
